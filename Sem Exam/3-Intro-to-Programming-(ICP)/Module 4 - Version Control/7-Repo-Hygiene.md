## ğŸ“Œ 1. Commit Message Guidelines
Write for humans, not robots.

- **Bad**: `fix`, `update`, `working on stuff`
- **Good**: `Fix login button crash on mobile div`
- **Format**:
  - `feat`: New feature
  - `fix`: Bug fix
  - `docs`: Documentation only
  - `style`: Formatting, missing semicolons...
  - `refactor`: Code change that neither fixes bug nor adds feature

## ğŸ§¹ 2. Hygiene Tips
- **Keep it clean**: Don't commit `.DS_Store` or `node_modules` (Use `.gitignore`).
- **Small commits**: Commit logical chunks of work.
- **Delete old branches**: Once merged, delete the feature branch.
